CRITICAL CONFIGURATION
================================================================================

CORS Configuration (.htaccess in public_html/)
───────────────────────────────────────────────
<IfModule mod_headers.c>
    Header set Access-Control-Allow-Origin "https://frontend.votredomaine.com"
    Header set Access-Control-Allow-Methods "GET, POST, OPTIONS"
    Header set Access-Control-Allow-Headers "Authorization, Content-Type, X-Requested-With"
    Header set Access-Control-Max-Age "3600"
</IfModule>

CHANGE: https://frontend.votredomaine.com to your actual domain!


React Router .htaccess (public_html/aos-frontend/.htaccess)
──────────────────────────────────────────────────────────
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>


API ENDPOINTS CREATED
================================================================================

Public Endpoints (No Authentication Required):
──────────────────────────────────────────────
POST   /wp-json/aos/v1/register
       Input: { email, nom, telephone, lieu_travail }
       Output: { success: true/false, message: "..." }
       Purpose: Register new user (checks whitelist)

POST   /wp-json/aos/v1/login
       Input: { email, unique_key }
       Output: { token: "...", user: {...} }
       Purpose: Generate session token

Protected Endpoints (Require Bearer Token):
───────────────────────────────────────────
GET    /wp-json/aos/v1/me
       Headers: Authorization: Bearer <token>
       Output: User profile data
       Purpose: Get authenticated user info

GET    /wp-json/aos/v1/demandes
       Headers: Authorization: Bearer <token>
       Output: List of user's requests
       Purpose: Show request history

POST   /wp-json/aos/v1/demandes
       Headers: Authorization: Bearer <token>
       Input: { type, montant, motif, ... }
       Output: { success: true }
       Purpose: Submit new request

Content Endpoints (Public):
───────────────────────────
GET    /wp-json/wp/v2/services      → List services with ACF fields
GET    /wp-json/wp/v2/actualites    → List news articles
GET    /wp-json/wp/v2/chiffres      → List statistics
GET    /wp-json/wp/v2/documents     → List documents


DATABASE TABLES
================================================================================

Three Tables Created Automatically:

1. aos_verified_employees (Whitelist)
   ├─ id (int)
   ├─ email (varchar)
   └─ added_date (timestamp)
   
   Purpose: Whitelist of employees allowed to register
   Populate: Admin adds emails before campaign launch

2. aos_adherents (User Accounts)
   ├─ id (int)
   ├─ email (varchar)
   ├─ nom (varchar)
   ├─ telephone (varchar)
   ├─ lieu_travail (varchar)
   ├─ unique_key (varchar)
   ├─ token (varchar)
   ├─ token_expiry (datetime)
   ├─ status (enum: pending, approved, rejected)
   └─ created_at (timestamp)
   
   Purpose: User accounts and authentication
   Workflow: pending → approved → can login

3. aos_demandes (Requests)
   ├─ id (int)
   ├─ adherent_id (int)
   ├─ type_demande (enum: aid, loan, vacation, autre)
   ├─ sujet (varchar)
   ├─ form_data (json)
   ├─ status (enum: pending, approved, rejected)
   └─ created_at (timestamp)
   
   Purpose: Track all submitted requests
   Data Stored: As JSON for flexibility
